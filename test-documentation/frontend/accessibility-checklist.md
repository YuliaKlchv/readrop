# Accessibility Checklist

## Keyboard navigation
- [ ] All interactive buttons and links are reachable by Tab.
- [ ] Form inputs have associated labels.
- [ ] The login and register forms are usable with keyboard-only input.
- [ ] Focus state is visible on buttons and inputs.

## Semantic markup
- [ ] Form fields use `label` elements correctly.
- [ ] Error messages use `role="alert"` where appropriate.
- [ ] Navigation elements use semantic `nav` and `aria-label` attributes.

## Visual contrast and readability
- [ ] Text contrast is sufficient on backgrounds.
- [ ] Buttons and links are readable and clearly distinguishable.
- [ ] Password strength and validation messages are visible.

## Helpful interactions
- [ ] `autoComplete` attributes exist on login/register fields.
- [ ] Action buttons include descriptive text.
- [ ] Guest redirect flows announce next steps clearly.
